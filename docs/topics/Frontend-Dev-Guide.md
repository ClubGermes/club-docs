# Frontend Developer Guide

## Запуск проекта

```bash
# Установка зависимостей
npm install

# Запуск dev-сервера
npm run dev
# или
yarn dev
```

Убедитесь, что настроен файл `.env` (или `.env.local`) для подключения к API.

## Рекомендации по разработке

1.  **Работа с данными:**
    *   При добавлении нового экрана: создайте файл в `src/queries/`, опишите модель, создайте `Entity` в компоненте и вызывайте через `collector.get`.
    *   **Не пишите `fetch` напрямую в компоненте.**

2.  **Стилизация:**
    *   Используется **TailwindCSS** + **DaisyUI**.
    *   Проверяйте `tailwind.config.js` и `styles.css` на наличие кастомных классов.

3.  **Дебаг:**
    *   **Аналитика:** Если графики "врут", проверяйте логику `createMonitor` в `Home.svelte`. Фильтрация происходит на клиенте.
    *   **API URL:** Проверяйте `src/stores/states.ts` — там хранятся базовые URL, подтягиваемые из ENV.

4.  **Performance:**
    *   Фильтрация в `Home.svelte` происходит по всему массиву логов. Будьте осторожны с тяжелыми вычислениями, используйте `debounce`.
